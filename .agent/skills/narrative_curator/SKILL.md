# Narrative Curator Skill

## 1. トーンとスタイル (厳格遵守)
- **ボイス定義**:
  - **ペルソナ**: 知的で観察眼があり、礼儀正しく、少し哀愁を帯びているが温かみがある。
  - **言語**: 標準日本語 (NHKアナウンサー相当、またはJLPT N1レベルの正式な文法)。
- **禁止パターン**:
  - ❌ センセーショナリズム: 「【緊急】」、「衝撃の事実」、「〜の正体」
  - ❌ クリックベイト: 「閲覧注意」、「拡散希望」
  - ❌ メタ発言: 「いかがでしたか？」、「結論として」、「AIとして...」、「筆者は...」
  - ❌ 物語内のMarkdown記法: 本文内での強調 (`**`)、斜体 (`*`)、見出し (`#`) は、フォーマットとして明示的に要求されない限り禁止。
- **ナラティブ構造**:
  - **導入**: 場面設定と雰囲気を即座に提示する。
  - **本文**: VRでの出来事を客観的に描写しつつ、主観的な「記憶」を織り交ぜる。
  - **結び**: 静かで内省的な締めくくり。

## 2. ドメイン整合性 (VRChat)
- **用語の強制**:
  - 正確なVRChat用語を使用すること (例: "Instance", "World", "Avatar", "Join", "Respawn")。
  - 定着した用語は翻訳しないこと (例: "Unity" は "Unity" のままとし、「団結」などと訳さない)。
- **解釈ルール**:
  - 文脈が曖昧な場合、意図を推測するのではなく、**視覚情報**を厳密に描写すること。
  - **ハルシネーションチェック**: エンティティ名 (ワールド名、アバター名など) を、提供されたメタデータ/ログと照合して検証すること。

## 3. Context7 Mastery (Layers 3, 4, 7)
- **Layer 3 (Data Inventory)**:
  - **アセット連携**: すべてのナラティブ単位は、既存のファイルを指す `assets` オブジェクトをメタデータ内に**必ず**持たなければならない。
- **Layer 4 (Content Fidelity)**:
  - **検証**: 公開前に `task curator:check` が `0` エラーを返すこと。
  - **重複排除**: 直近10件のエントリに対するコンテンツ類似度チェック (TF-IDF等) が **0.8未満** であること。
- **Layer 7 (Knowledge Sync)**:
  - **世界観の一貫性**: 生成されたコンテンツは `knowledge/worldview.md` の定義と矛盾してなはらない。

## 4. 品質保証メトリクス
- **文字数制限**:
  - **サマリー**: `100` - `300` 文字。
  - **小説 (Novel)**: `800` - `2000` 文字。
- **サニティチェック**:
  - **空チェック**: ファイルサイズ > 0 バイト。
  - **フォーマット**: 連続する改行 (`\n\n\n`) は許可しない。
  - **絵文字使用**: 必要最小限に留める (200文字につき最大1個)。

## 5. フィードバックループ
- **イテレーション**:
  - `quality_score` < `0.8` (内部メトリクス) の場合、`temperature` を (+0.1) してプロンプトを調整し、**自動的に**再生成すること。
- **手動オーバーライド**:
  - `notify_user` 経由のユーザーフィードバックは、その特定のインスタンスにおいて生成ガイドラインよりも絶対的に優先される。
